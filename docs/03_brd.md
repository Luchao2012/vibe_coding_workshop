# Business Requirements Document (BRD)

## Objective

Build an automated chatbot to handle repetitive ACC customer inquiries, reducing administrative workload and improving response times for prospective and current members.

## Target Users

**Primary:**
- Prospective members exploring ACC
- Current members with routine questions
- Community members seeking information

**Secondary:**
- ACC staff (benefit from reduced repetitive work)

## Business Value

**Quantitative benefits:**
- Reduce staff time spent on repetitive inquiries by ~60%
- Provide instant responses (vs. hours/days currently)
- Enable 24/7 availability

**Qualitative benefits:**
- Improve member experience (faster, consistent answers)
- Free staff to focus on strategic initiatives
- Scalable as ACC grows (no linear increase in staff needed)
- Consistent, accurate information delivery

## Success Metrics

**For MVP/Workshop:**
- Chatbot successfully answers 10+ common questions about ACC
- Response time < 5 seconds per query
- Answers are factually grounded in website content (no hallucinations)

**Future metrics (post-workshop):**
- 70% of inquiries resolved without human intervention
- User satisfaction rating > 4/5
- Reduction in staff inquiry response workload

## Constraints

**Legal/Privacy:**
- No personal data collection (names, emails) in MVP
- Must use publicly available information only
- Comply with website crawling best practices (robots.txt)

**Technical:**
- Must use free/low-cost tools suitable for workshop
- Local deployment (no production infrastructure)
- Must work on personal laptops

**Time:**
- Must be demonstrable in workshop setting (2-3 hours)

**Access:**
- Only public ACC website content (no internal systems)

## Out of Scope

- Payment processing
- User authentication
- Booking/registration systems
- Multi-language support (English only for MVP)
- Mobile app
- Integration with ACC's production systems
- Staff training portal
